SelfHost Firefox Sync Server

Host Browser Paswords history and Bookmarks on Your RaspberryPi

Sep 01, 2022
3 minute read

Mozilla browser can be completelly selfhosted. Comes in two servers a sync server which will sync passwords , browse history ,bookmarks and a acounts server here we’ll see how to self host the sync server using docker The dockerfile builds https://github.com/mozilla-services/syncserver Which is an all-in-one package for running a self-hosted Firefox Sync server. It bundles the “tokenserver” project for authentication and the “syncstorage” project for storage Firefox marked the project as deprecated and moved to storageServer to provide a sync server the will serve as a backup solution with a “distributed database to store the data secure” as the curent sync server was not created as a backup solution in mind and the dB could be reseted at times. Let me clarify that a bit source Your ff data leaves in every device you use it amd syncs with the server . so the server is always to the newest copy of your data.periodicaly mozilla shuts down servers.that in most cases is not a problem if you have an active device as their new server will sync to your newest version of your data from that device.In an extreme scenario when you only have one active device amd havent used it for a.long time amd you uninstall your firefox.there qell then there.is a chance.you’ll loose your server db. that said lets clarify that when self hosting firefox this is not a problem as you are not going to reset your raspberry amd you are keeping backuos of your db . right?
anyway their new sync server is more than wellcome and Until that becomes stable and a docker image becomes available we can still continue use the curent server *at our own risk. You can track down the issue here

version: "3.2"

services:
  firefox-syncserver:
    image: crazymax/firefox-syncserver:latest
    container_name: firefox_syncserver
    ports:
      - target: 5000
        published: 5000
        protocol: tcp
    volumes:
      - "/local/filepath/for/your/data:/data"
    env_file:
      - "./firefox-syncserver.env"
    restart: always

volumes:
  firefox-syncserver:

Notice the data volume that you’ll have to map to a dir to your machine And the .env file in the current dir

Data dir will store the dB with the sync data

The firefox-syncserver.env will have to be created Use the template from here

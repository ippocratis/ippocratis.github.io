Knockd

A port securing method

Mar 30, 2021
2 minute read

knockd

In order to harden our raspberry server we will install some services. Not too techy but strong. Knockd is one of them. Knockd listens on ports and waits for a specific sequence in them. When this happens it executes a command from its configuration file. Here we will execute an openSSH command and open the ssh port on our firewall. So we do not need to have the ssh port permanently open, except when requested with the right combination.

First copy the name of the Interface in which we do ssh and knocking

ifconfig

install knockd

apt install knockd

configure the config file to request 3 ports which should be called within 5sec. Within This time we should also call ssh. To open the ssh port and delete our ip after 15sec without Log us out but require a knockd after that time

nano /etc/knockd.conf

[options]

[openSSH]
        sequence = xxxx, xxxx, xxxx
        seq_timeout = 5
        command = / sbin / iptables -A INPUT -s% IP% -p tcp --dport 22 -j ACCEPT
        tcpflags = syn
        cmd_timeout = 15
        stop_command = / sbin / iptables -D INPUT -s% IP% -p tcp --dport 22 -j ACCEPT

activate the service

In debian knockd cannot start at boot. so

nano /lib/systemd/system/knockd.service

and add at the end

[Install]
WantedBy = multi-user.target

Also in the unit section to make sure that the service will start after the networking is restored during boot add

After = network-online.target
Wants = network-online.target

otherwise if it starts before the wlan0 interface connects to the inernet knockd will fail

activate the service

systemctl enable knockd.service

watch it live

knockd -D -v

start the service

service knockd start

From this point onwards in the host in order to be able to enter raspberry with ssh we must first knock

knock -v IP port1 port2 port3

where

ip = rpi ip

ports = configured xxxx pors

now we can enter with

ssh user @ ip

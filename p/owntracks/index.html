Dockerised Owntracks recorder

  • What is it:

OwnTracks allows you to keep track of your own location. You can build your private location diary.

  • how:

An android app sends location in fixed intervals to the owntracks server

  • DDNS:

You’ll need a free subdomain name and redirect that to your localhost port owntracks is running Get one from freedns,ydns,duckdns etc You’ll also need a cronjob to auto update you ip

  • Caddyfile:

Reverse proxy with basic auth and auto-ssl

version: '3'

services:

  otrecorder:
    container_name: ot-recorder
    image: jessestuart/owntracks
    environment:
      - OTR_PORT=0  # disables MQTT
    volumes:
      - /etc/localtime:/etc/localtime:ro
      - /run/media/ippo/TOSHIBA/owntracks-recorder/config:/config
      - /run/media/ippo/TOSHIBA/owntracks-recorder/store:/owntracks/recorder/store
    ports:
      - 8083:8083
    restart: unless-stopped

Note that the jessestuart/owntracks image was used as owntracks provide no arm64 image also there are build instructions but with lots of dependencies

install photoview image gallery

4 minute read

photoview is lightweight dockerised photo gallery with face detection and auto library sync with modern design and useful features and an optimized mobile view as a web app it can easily replace proprietary paid closed source applications.

reqirements

python for the docker-compose installation on raspberry since the normal repos do not work on it docker and if we want to expose it outside the local network port forwording, ddns provider, reverse proxy server, encryption

Dietpi has its own scripts for installing programs and services so in this distro, from dietpi-software and additional-software we can install python and required dependencies if we use another distro

install python

sudo apt install python3 idle3

install dependencies

sudo apt-get install -y libffi-dev libssl-dev sudo apt-get install -y python3 python3-pip sudo apt-get remove python-configparser

install docker/docker-compose

install docker with official script

curl -fsSL https://get.docker.com -o get-docker.sh

sudo sh get-docker.sh

and docker-compose through python

sudo pip3 -v install docker-compose

finally install photoview with docker-compose

mkdir photoview

cd photoview

nano docker-compose.yml

version: "3"

services:
  db:
    image: mariadb:10.5
    restart: always
    environment:
      - MYSQL_DATABASE=photoview
      - MYSQL_USER=photoview
      - MYSQL_PASSWORD=photosecret
      - MYSQL_RANDOM_ROOT_PASSWORD=1
    volumes:
      - db_data:/var/lib/mysql

  photoview:
    image: viktorstrate/photoview:edge
    restart: always
    ports:
      - "8000:80"
    depends_on:
      - db

    environment:
      - PHOTOVIEW_DATABASE_DRIVER=mysql
      - PHOTOVIEW_MYSQL_URL=photoview:photosecret@tcp(db)/photoview
      - PHOTOVIEW_LISTEN_IP=photoview
      - PHOTOVIEW_LISTEN_PORT=80
      - PHOTOVIEW_MEDIA_CACHE=/app/cache

      # Change This: The publicly exposed url
      # For example if the server is available from the domain example.com,
      # change this value to http://example.com/
      - PHOTOVIEW_PUBLIC_ENDPOINT=http://localhost:8000/

      # Optional: To enable map related features, you need to create a mapbox token.
      # A token can be generated for free here https://account.mapbox.com/access-tokens/
      # It's a good idea to limit the scope of the token to your own domain, to prevent others from using it.
      # - MAPBOX_TOKEN=<YOUR TOKEN HERE>

    volumes:
      - api_cache:/app/cache

      # Change this to the directory where your photos are located on your server.
      # If the photos are located at `/home/user/photos`, then change this value
      # to the following: `/home/user/photos:/photos:ro`.
      # You can mount multiple paths, if your photos are spread across multiple directories.
      - ./photos_path:/photos:ro

volumes:
  db_data:
  api_cache:

modify docker-compose.yml

edge is the unreleased beta edition .ther options are nighty and 2 for stable.

ports: - “8000:80”

instead of 8000 you can choose any available port on host

PHOTOVIEW_PUBLIC_ENDPOINT=http://localhost:8000/

replace with your domain if you want to expose to public thumbnail previews are only available from this adress

MAPBOX_TOKEN=

uncomment # grab token from https://account.mapbox.com/access-tokens/create after signing up on this great Open source map api service make sure to add yoour domain tthere so that only that can acess your token.

./photos_path:/photos:ro replace ./photos_path with the full path of photos folder on host . path can be on an external hard drive

install

docker-compose up -d

Select external storage for cache volume

The app will create a relatively large size of thumbnails in the docker volume of api_cache so it is probably better to move the cache to external hard drive This can be done either by mounting - bind a folder of the hard drive in the container or directly to the docker-compose which is more elegant

replace

volumes: - api_cache:/app/cache

with

volumes: - /path/to/external/drive/folder:/app/cache

delete

api_cache:

from

volumes: db_data: api_cache:

and leave

PHOTOVIEW_MEDIA_CACHE=/app/cache

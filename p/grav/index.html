Deploy Grav to Heroku Paas

Mar 20, 2021
3 minute read

note that heroku is about to change their charging policy and their free dyno tier will be removed. therefor this deploy is no longer valid as.a.no cost option

Grav is a self-hosted content management system (CMS) written in PHP. You can create web pages with it and modify their content online. It has an admin panel from which you add and modify your content it is rich in plugins and functions very easy to use for the simple user with several modern and classic templates and themes for everyone also has skeletons that are 100% ready to use sites that all you have to do is adapt their texts to your needs.

we will use docker-compose with the grav files mapped in the current directory so that we have them in a folder in the host for immediate modification and easy copying in the working heroku folder. and yeah we’ll deploy it to heroku Heroku is a platform as a service (PaaS) service with which we create and run applications in the cloud. We will use it to upload our grav site and run it from heroku as opposed to selfhost it on the pi to avoid exposing our public ip as it is a service that will be open to the general public.

Create the docker container

mkdir grav

cd grav

git clone https://github.com/getgrav/docker-grav.git

cd docker-grav

mkdir grav

nano docker-compose.yml

change port if you like

docker-compose up -d

launch localhost:port/admin

create your account

** skeleton **

Skeletons

has ready-to-use sites

Try the demo in each one to watch it live Download the skeleton you want From downloaded file extract the user folder Delete the contents of ~/grav/docker-grav/grav/user folder and keep the hidden .git folder replace the user folder with the files from the skeletons user folder we extracted

give rights

chown -R www-data:www-data /root/grav/docker-grav/grav/

To see your site visit

localhost:port in the browser

Admin panel

The admin panel is located at

localhost:port/admin

The structure of the site is arboreal Starts from the root directory / and adds folders and subfolders with pages inside From the pages tab we delete, add or modify the existing pages In content tab we write our text and title of the page using markdown and in the options we activate it while in the media field are the photos

Host to heroku

We need the javascript npm package manager which is installed along with nodejs Download according to our disstro from here

For debian based we need the binary from here

In ubuntu

curl -fsSL https://deb.nodesource.com/setup_15.x | sudo -E bash - sudo apt-get install -y nodejs

install heroku cli

npm install -g heroku

signup for a free acunt at https://signup.heroku.com/

heroku signin

We create a working directory for heroku and copy the grav files inside

mkdir heroku-gtav

cd heroku-grav

cp ~ /grav/docker-grav/grav ~/heroku-grav

git init

create the app

heroku create

chech how heroku has named your app. We can change this name from https://dashboard.heroku.com/apps/app-name/settings either case we define heroku remote with the name of the app

heroku apps

heroku git: remote -a yourappname

deploy your site

git add .

git commit -am "any_text"

git push heroku master

launch it

heroku open
